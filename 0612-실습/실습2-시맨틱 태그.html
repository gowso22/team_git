header div 태그를 사용하고 아이디를 header로 사용하고 있다. 시맨틱 태그 header를 사용하고 있다
main div 태그를 사용하고 아이디를 container로 사용하고 있다. div 태그로 시맨틱 태그 section을 감싸서 사용하고 있다.
footer div 태그를 사용하고 아이디를 footer로 사용하고 있다. 시맨틱 태그 footer를 사용하고 있다

시맨틱 태그를 사용해야하는 이유

  1. 1. 검색 엔진 최적화

    시맨틱 태그를 사용하면 웹 페이지의 구조와 콘텐츠를 검색엔진이 더 잘 이해 할 수 있다.
    검색 엔진은 시맨틱 태그를 사용해서 문서를 분석하고, 콘텐츠의 의미와 구조를 파악해 검색 결과에 반영 할 수 있다.

  2. 2. 웹 접근성

    시맨틱 태그를 사용하면 웹 페이지의 구조가 명확해지므로 스크린 리더와 같은
    보조 기술을 사용하는 사용자에게도 정보를 잘 전달할 수 있다.
    시각 장애인이나 저시력자들은 시맨틱 태그를 통해 웹 콘텐츠를 이해하고 탐색 할 수 있다

  3. 3. 유지보수와 가독성

    시맨틱 태그는 문서의 구조를 명확하게 표현하기 때문에 코드의 가독성을 향상 시킨다.
    시맨틱 태그를 사용하면 개발자들이 문서의 의미를 더 잘 이해할 수 있으므로
    협업과 유지 보수 과정에서도 도움이된다.

  4. 4. css 스타일링

    시맨틱 태그는 기본적으로 의미를 담고 있으므로 css 스타일링을 적용할 때도 의미에 맞게
    스타일을 적용하기 용이하다. 시맨틱 태그를 사용하면 불필요한 클래스나 id를 추가로 작성하지 않아도 된다.

  5. 5. 향후 웹 표준 호환성

    시맨틱 태그를 사용하면 향후 웹 표준 호환성을 유지하고 새로운 기능과 업데이트에
    더욱 적극적으로 대응할 수 있다.
